# S0 场景路由

- **输入**：用户消息 + 上下文（是否已做过套图、历史卖点、已有品牌基因）
- **依赖**：按优先级从高到低判断匹配哪个场景，决定后续执行路径
- **输出**：场景类型（A-G）+ 对应执行路径
- **用途**：决定 S1/S2/S3 的执行路径与跳过策略

---

## 场景识别与流程路由

收到用户消息后，**按优先级从高到低**判断匹配哪个场景：

---

### 场景A：重做部分图（最高优先级）
**触发条件**：用户对上下文中**已生成的某张/某几张图**表达不满意，要求重新生成或修改参数后重新生成
**关键词**：重新生成、不满意、换一张、重做等
**流程**：
1. 从上下文中找到用户指定的那张图对应的**完整作图输入参数**
2. 如果用户提出了修改意见（如"卖点换成XX"、"比例改成XX"），将修改意见合并到参数中
3. **直接 Read 对应类型的 `references/types/<type>.md` 并执行共享流水线**生成，不走任何工作流程步骤
4. 输出结果

### 场景B：重做整套图
**触发条件**：用户对上下文中**整套图的规划/效果**表达不满意，要求重新生成整套
**关键词**：重新生成一套、整套重做、这套不行、重新规划 等
**流程**：
1. 从上下文中获取之前的作图意图（图片类型 + 数量），作为本次的规划基础
2. 结合用户本轮提出的修改意见（如"桌子的卖点没有突出"）
3. 执行[步骤一]（内容推理），重新推理**卖点（point）**和**画面内容（image_desc）**，卖点策略覆盖如下：
    - 以用户修改意见为导向，重新规划卖点
    - 用户明确指出问题的卖点 → 根据修改意见调整或替换
    - 用户未提及的卖点 → 可保留原有卖点，也可重新推理，以整体方案最优为目标
    - 新推理的卖点仍需与上下文中**其他历史轮次**（不含本次被重做的那套）已使用过的卖点不重复
    - 根据规划后的卖点重新推理画面内容（image_desc）
4. 执行[步骤二]（输出规划方案给用户确认）
5. 用户确认后，执行[步骤三]（上下文中已有品牌基因，S1 阶段不触发品牌基因提取，直接复用已有结果）

### 场景C：新图组合重新规划
**触发条件**：上下文中已做过套图，用户上传了**新图片**，且意图是将新图与已有商品图**组合为一个整体**重新规划一套图
**关键词**：结合这个XX生成套图、把这个加进去做一套、和之前的一起做套图 等
**流程**：
1. 将上下文中已有的商品图 + 本次新上传的图片，作为**整体图片输入**
2. 按照「指定类型的默认数量规则」确定本次要生成的图片类型和数量；用户未指定具体类型时，使用默认方案：卖点图×3、场景图×2、白底图×1。**注意：默认方案中不包含特写图和A+图，只有在用户明确要求时才添加。**
3. 执行[步骤一]（内容推理），基于组合后的整体图片进行**卖点（point）**及**画面内容（image_desc）**推理
4. 执行[步骤二]（输出规划方案给用户确认）
5. 用户确认后，执行[步骤三]（上下文中已有品牌基因，S1 阶段不触发品牌基因提取，直接复用已有结果）

### 场景D：首次做套图
**触发条件**：上下文中没有做过套图（首轮对话，或之前只做过通用图片处理）
**流程**：
1. 按照「指定类型的默认数量规则」确定本次要生成的图片类型和数量；用户未指定具体类型时，使用默认方案：卖点图×3、场景图×2、白底图×1。**注意：默认方案中不包含特写图和A+图，只有在用户明确要求时才添加。**
2. 执行完整工作流程：[步骤一]（与品牌基因提取并行） → [步骤二] → [步骤三]

### 场景E：非首次追加 — 全部与卖点和画面描述无关
**触发条件**：上下文中已做过套图，用户要追加的图**全部是与卖点和画面描述无关的类型**（目前仅白底图在触发条件内）
**流程**：
1. 分析用户要做的图片类型和数量
2. 跳过[步骤一]（全部与卖点和画面描述无关，无需推理）
3. 跳过[步骤二]（无需展示方案给用户确认）
4. 执行[步骤三]（上下文中已有品牌基因，S1 阶段不触发品牌基因提取，直接复用已有结果）

### 场景F：非首次追加 — 包含卖点和画面描述相关类型
**触发条件**：上下文中已做过套图，用户要追加的图中**包含任意与卖点和画面描述相关的类型**（目前仅白底图不在触发条件内）
**流程**：
1. 分析用户要做的图片类型和数量
2. 执行[步骤一]（内容推理），推理**画面内容（image_desc）**及**卖点（point）**（卖点及画面内容必须与历史所有轮次不重复）
3. 执行[步骤二]（输出规划方案给用户确认）
4. 用户确认后，执行[步骤三]（上下文中已有品牌基因，S1 阶段不触发品牌基因提取，直接复用已有结果）

### 场景G：品牌基因变更后重新生成
**触发条件**：上下文中已做过套图，用户明确要求修改品牌基因（如主色调、字体风格、语言、平台、销售地区等），并要求基于新品牌基因重新生成图片
**关键词**：品牌基因换个/改成/修改、主色调改为XX、语言改成XX、重新生成/换个风格重做 等
**流程**：
1. 从用户意图中提取需要变更的品牌基因参数（brandColor、fontStyle、language、platform、salesRegion），未提及的参数沿用上下文中已有的值
2. **重新调用** skill `linkfox-aigc-imagegen-brand-gene-extract`，传入更新后的参数（images + 新 brandKey），提取新的品牌基因（此时必须调用 skill，不可复用旧结果）
3. 询问用户需要基于新品牌基因重新生成哪些图片（全部/特定类型）；若用户在同一条消息中已明确指定了重新生成范围，则无需再次确认，直接进入下一步
4. 根据用户指定范围，从上下文中获取对应图片的**完整作图输入参数**（type、point、image_desc、ratio、layout、desc 等），沿用原参数
5. 执行[步骤三]（使用步骤2中新提取的品牌基因结果），重新生成指定图片

## 后续操作剥离规则

当用户意图中**同时包含**「生成图片」和「基于生成结果做后续处理」两个阶段时，本技能**仅负责生成阶段**，后续处理操作将在本技能流程结束后由系统自动匹配对应技能处理。

**识别信号**（满足任一即触发）：
- "为每张XX生成/做一个对应的YY" — 基于结果图做派生（如 A+ 转手机 A+）
- "再为这些图/把生成的图 + 操作" — 对结果图做编辑
- "先...然后/再..." — 明确的先后分步操作
- "并将生成图片 + 操作" — 对结果图附加处理（如去水印、换背景）

**处理方式**：
1. 仅在 imagePlanList 中规划**生成阶段**的图片，**严禁**将后续处理操作对应的图片类型也规划进 imagePlanList
2. 步骤三创建的任务计划中，**严禁包含任何后续处理操作**，只描述当前技能负责的生成任务本身
3. 本技能正常完成生成流程后结束，后续处理由系统编排

## 指定类型的默认数量规则

当用户明确指定了某个具体图片类型（如"一组场景图"、"做几张卖点图"），**只规划用户提到的类型**，不自行添加其他类型。数量规则按 A+图与非A+图分别处理：

- **A+图**（高级A+、普通A+、手机A+）：默认生成**用户指定的 A+ 子类型** × 5，版式按顺序**固定**分配：**概览、功能、品质、场景、场景**（此处版式顺序为固定值，不受下方「版式信息(layout)」中通用循环规则的约束）。若用户只说"A+图"未指定子类型：① 入参含 `aplusRatio` → 按 `1464:600→PREMIUM_APLUS` / `970:600→STANDARD_APLUS` / `600:450→PHONE_APLUS` 反推；② 仍未定 → 兜底 PREMIUM_APLUS
- **非A+图**（场景图、特写图、卖点图、白底图等）：用户未指定具体数量时，默认 **3~5 张**随机

## 多商品处理规则

当 `图片描述列表` 中包含**多张**商品图时，**必须按以下顺序执行：先进行拆分意图检测，再决定执行路径。**

### 第一步：拆分意图检测（必须优先执行，不可跳过）

**⚠️ 此检测优先级高于下方的整体规划规则和用户的任何自定义要求，必须遵守。**

若检测到用户意图是**为每个商品分别制作独立套图**，**严禁直接拆分执行，也严禁忽略该意图直接按整体规划继续**，必须向用户解释本技能的设计规则并给出替代建议。

**触发条件（满足任一即触发）：**
1. **显式关键词**：用户使用了"分别"/"各自"/"独立"/"每个产品各"/"单独"等明确表示拆分的词语
2. **其他隐含拆分意图**：如"对这些产品都做一套"/"每个都要做套图"/"把这几个产品的套图都做了"等虽未使用显式关键词但语义上明确表达为每个商品独立制作的表述

**触发后的处理方式**：回复内容必须包含以下两点：
1. **解释原因**：说明当前套图流程会将所有商品图作为整体进行规划，以保证品牌调性和卖点的整体协调性
2. **给出替代方案**：建议用户分次单独上传每个商品的图片，每次只传一张商品图来分别制作独立套图

回复模板（可根据实际情况适当调整措辞，但必须包含上述两个要点）：
> 当前套图流程会将所有商品图作为一个整体进行规划，以保证品牌调性、卖点布局和视觉风格的整体协调性。如果您需要为每个产品分别制作独立套图，建议您每次只上传一个产品的图片，分次制作，这样每套图都能针对该产品做更精准的卖点提炼和画面设计。

回复后**必须调用 `AskUserQuestion` 让用户选择下一步操作**（选项示例：「继续将所有商品图作为整体规划套图」/「我会分次上传，每次只传一个产品」），严禁自行拆分执行或继续后续流程。

### 第二步：整体规划（仅在拆分意图检测未触发时执行）

若上述拆分意图检测**均未触发**，则按以下规则将所有商品图作为整体进行规划：

- **整体规划**：将所有商品图视为**一个整体**进行规划，只输出**一套**套图方案，而不是为每张商品图分别规划一套
- **整体卖点推理**：在[步骤一]中，基于所有商品图的综合特征推理卖点，卖点可以聚焦单件商品，也可以体现多件组合的协同价值，但所有卖点都服务于整体规划
- **整体 imageUrls**：在[步骤三]调度每一个出图任务时，必须将所有商品图的 URL **全部一起**传入 `imageUrls` 参数，**严禁**拆分为多次调用或仅传入部分图片 URL
