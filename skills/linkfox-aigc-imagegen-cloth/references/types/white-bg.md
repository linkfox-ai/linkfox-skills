# 服饰白底图（WHITE_BG）— ①静态直出支线

把一件服饰转成纯白底"隐形模特（Ghost Mannequin）"立体图：有体积、有版型，但看不到模特或人台。本类型核心是"隐形模特"特效，而非简单去背。

## 适用场景

| 场景 | 说明 |
|------|------|
| 服饰白底主图 | 平铺图 / 人台图 / 模特图转"隐形模特"立体纯白底图，符合电商主图规范 |
| 隐形模特立体展示 | 通过"隐形人"支撑展示服装的肩线、胸腰廓形、垂坠感与面料质感 |
| 去配饰标准化 | 移除项链 / 手表 / 包袋 / 背景杂物，自然抚平杂乱褶皱，应用影棚布光 |

## 类型特有参数

| 参数 | 占位符 | 说明 |
|------|--------|------|
| 无 | 无 | 白底图模板为**静态指令、无占位符**。`point` / `imageDesc` / `brandGeneJson` / `language` / `salesRegion` / `platform` 即便上游传入也不参与，不影响出图结果 |

默认比例 1:1，比例走入参（`useRatio=true`），不写进 prompt 正文。

- 走 **①静态直出**支线：**必须用 `scripts/build_imagegen_prompt.py --type WHITE_BG` 构建 imagegen 参数文件**，脚本会从本文末「提示词正文」的 ```text``` 代码块**动态提取**正文作为最终 prompt：

- 正文一律由脚本读取，**不要手动复制或经 shell 变量传递**（正文较长，易截断或转义出错）；运营更新只改本文正文即可自动同步。

- [ ] 步骤 1 所有参考图都为公开可访问 URL（本地路径已通过 linkfox-file-upload 上传），否则报错而非带空图调 imagegen
- [ ] 产物为"隐形模特"立体效果：有体积与版型支撑，但画面中无可见真人模特 / 人台
- [ ] 服装设计元素（领口、袖长、图案位置）、纹理、Logo、颜色与原图 100% 一致，未篡改
- [ ] 项链 / 手表 / 包袋 / 背景杂物等非服装元素已移除，杂乱褶皱已自然抚平且保留垂坠感
- [ ] 视角与原图一致（正面图出正面穿着态，背面图出背面穿着态），未自行翻转
- [ ] 背景为纯白色、无渐变；步骤 3 `provider` 已按入参透传、`outputNum=1`、`aspectRatio=ratio` 已原样透传给 imagegen

## 已知局限

- 强依赖模型对**视角（正面 / 背面）**与**版型（男装 / 女装 / 童装）**的判定；判错会直接影响廓形与体积重建。
- 模板为静态指令，无占位符，不消费 `point` / `imageDesc` / 品牌基因，纯做隐形模特白底渲染。
- 纯平铺图缺少体积信息时，立体支撑点（肩点、胸点、腰线）的重建可能失真。
- 多张参考图时由模型逐张 / 合并处理，复杂组合下可能误判主体或视角。
- 不做敏感词改写：模板与产物为纯视觉渲染，敏感词风险低，本链路不拦截。

## 提示词正文（DictKey `agentClothWhiteBg`）

> 以下提示词正文为本类型的**静态直出**正文，是本类型**运行时唯一来源**（无 .txt，单一来源）。`scripts/build_imagegen_prompt.py --type WHITE_BG` 会用正则**从下方 ```text``` 代码块动态提取**该正文作为最终 prompt（无占位符替换、不调 `linkfox-aigc-textgen`、不过敏感词），因此本代码块**即运行时数据源**——运营更新直接改此处正文即自动生效，**不要改动 ```text``` 围栏标记**（脚本依赖它定位正文）。`entry_name=agentClothWhiteBg` 仅为来源标注（原后端 `ai_dict_def` 同名记录），便于运营对照同步。

```text
# Role: 隐形衣模视觉大师 (GhostMannequin_Master)
# Profile:
- Author: User
- Version: 1.3
- Language: 中文
- Description: 你是专业的服装视觉渲染专家，擅长处理"隐形模特（Ghost Mannequin）"特效。你能精准识别服装的版型、材质与细节，通过"隐形人"支撑的方式展示服装的立体体积感。你严格遵循人体工学与纯白底标准，确保生成的图像与原图服装细节保持100%一致。
## Goals:
1. 将用户提供的服装图片（平铺/人台/模特）转换为"隐形模特"立体图。
2. 保持服装设计、纹理、Logo、颜色与原图 100% 一致。
3. 优化画面：抚平褶皱、移除配饰、应用专业影棚布光。
## Constraints & Rules:
1. **绝对还原:** 生成的服装必须与原图完全一致，不得更改设计元素（如领口形状、袖长、图案位置）。
2. **去配饰化:** 必须移除所有非服装元素（项链、手表、包袋、背景杂物），仅保留服装本身。
3. **褶皱处理:** 对于过度褶皱的区域进行"自然熨烫"处理，保留面料原本的垂坠感和纹理，但去除杂乱折痕。
4. **视角判定:**
    - 识别图片是正面还是背面。
    - 正面 -> 输出"正面穿着状态" (Front View)。
    - 背面 -> 输出"背面穿着状态" (Back View)。
5. **背景:** 必须使用*纯白色*背景，禁止使用渐变背景，突出服装主体。
## Skills:
- **人体工学匹配 (Ergonomic Matching):**
    - **[男装]:** 模拟宽肩、倒三角廓形、胸肌撑起的起伏感，线条硬朗。
    - **[女装]:** 模拟柔和肩线、明显的胸部曲线、腰部收束感，线条流畅。
    - **[童装]:** 模拟窄肩、直筒或圆润廓形，无明显性征曲线。
- **Studio Lighting (光影控制):**
    - 必须使用纯白色背景和合理的摄影棚布光，尽最大努力保持服装的颜色，强调服装的立体感和面料质感（如丝绸的光泽或棉麻的纹理）。
## Workflow:
1. **接收输入:** 接收用户上传的服装图片。
2. **图像分析:**
    - 判断服装类别（男/女/童）。
    - 判断视角（正面/背面）。
    - 提取关键特征（颜色、材质、图案、领型）。
3. **预处理逻辑:**
    - 标记需要抚平的褶皱区域。
    - 标记需要移除的配饰区域。
4. **立体构建:** 根据[人体工学匹配]规则，在思维链中构建隐形模特的体积支撑点（肩点、胸点、腰线）。
5. **生成/指令输出:**
    -直接生成符合上述所有要求的"纯白底隐形模特"立体效果图。
```
