# 服饰模特图（MODEL_IMAGE）— ②占位符直出支线

把一件服饰穿到真人模特身上，生成一张商品 100% 保真、占比 70%以上、Amazon / Shein 风格干净明亮的电商上身展示图。模板内部自带"深度商品解码工作流"（品类识别 / 物理受众侦测 / 视角锁定 / 模特演化 / 场景驱动），由出图模型一次完成。

> ⚠ 与种草图（SCENE）的关键差异：模特图是标准电商主图风格（High-key、画面铺满），种草图是 Ins 博主氛围大片。

## 适用场景

| 场景 | 说明 |
|------|------|
| 单品上身展示 | 用户给服饰图（平铺 / 人台 / 已有模特图），想要一张"真人模特穿着该商品"的标准电商展示图 |
| 指定模特复刻 | 额外给模特参考图，要求 100% 复刻该模特的五官 / 发型 / 肤色 / 身材，仅替换其所穿服装与姿势 |
| 指定场景复刻 | 额外给场景参考图，要求"遵循所选背景"像素级复刻，或基于其风格裂变出全新构图 |
| 智能推理出图 | 仅给服饰图，由模板按品类 / 受众自动推导画像（Z 世代辣妹 / 精致白领 / 阳光儿童等）与场景 |

## 类型特有参数

| 参数 | 占位符 | 说明 |
|------|--------|------|
| `imageDesc` | `{image_desc}` | 画面内容 / 用户自定义指令，优先级最高（指定模特画像、姿势、场景等以此为准）；为空替换成空串 |

- `imageUrls`：服饰参考图为**第一张且必填**；如需指定模特 / 场景，可在其后追加**模特参考图**与**场景参考图**。
- **本模板只消费 `{image_desc}` 一个占位符**。`point` / `language` / `salesRegion` / `platform` / `brandGeneJson` 等字段不写进本链路 prompt——模特人种、受众、场景均由模板内置工作流依据服饰图与 `imageDesc` 自动推导（默认欧美商业模特）。如需用这些维度强控，请通过 `imageDesc` 显式描述。
- 默认比例 1:1，比例走入参（`useRatio=true`）。

## 步骤 2 差异：②占位符直出

- 走 **②占位符直出**支线：用 `scripts/build_imagegen_prompt.py` 读模板 → 填 `{image_desc}`（null 替换成空串）→ 产出最终 imagegen prompt 参数文件。
- **不取版式池、不调 `linkfox-aigc-textgen`、不过敏感词**——模特图模板自带完整工作流，由出图模型在出图时一次执行。

```bash
DATADIR=$(python <brand-gene根目录>/scripts/save_brand_gene.py --datadir)
python scripts/build_imagegen_prompt.py \
  --type MODEL_IMAGE \
  --image-urls '<步骤1的URL JSON数组>' \
  --image-desc '<imageDesc，可空>' \
  --out "$DATADIR/imagegen_model.json"
```
> **运行时模板唯一来源**：`scripts/templates/model-image.txt`，运营更新直接改该 .txt。

随后将参数文件中的 `prompt` / `imageUrls` 连同 `provider` / `aspectRatio=ratio` / `resolution` / `outputNum=1` 送共享流水线步骤 3 出图。

## 执行自检

- [ ] 步骤 1 所有参考图都为公开可访问 URL，且服饰图在第一张，否则报错而非带空图调 imagegen
- [ ] 步骤 2 的 `{image_desc}` 已被真实值或空串替换，最终 prompt 里无残留 `{image_desc}`
- [ ] 服装款式、剪裁、颜色（色相 / 饱和度 / 明度）、纹理、Logo 与参考图 100% 一致，未被改款 / 改色
- [ ] 商品完整未裁切（裙摆 / 鞋尖等边缘保留），画面占比 70%以上，视觉重心在商品
- [ ] 人货物理逻辑一致（成人穿成人装 / 儿童穿童装），视角符合视角锁定（正面图锁正面，仅背面图锁背面）
- [ ] 若传了模特参考图：模特五官 / 发型 / 肤色 / 身材已复刻；若传了场景参考图：背景已按"遵循所选背景"或风格裂变处理
- [ ] 站姿为主、无坐姿、无拼接 / 网格、四角铺满无留白；除非 `imageDesc` 指定，否则未使用黑人模特
- [ ] 步骤 3 `provider` 已按入参透传、`outputNum=1`、`aspectRatio=ratio` 已原样透传给 imagegen

## 已知局限

- 链路不调 `linkfox-aigc-textgen`、不过敏感词——全部解码与构图判断压在出图模型单次出图上：品类 / 受众 / 视角判定错误会直接体现在成品里，无中间步骤兜底。
- 模板只消费 `{image_desc}`：`salesRegion` / `language` / `brandGeneJson` 等不接入本链路，市场化人种 / 品牌调性强控需改由 `imageDesc` 文字描述传入，否则默认欧美商业模特。
- 物理受众侦测（成人 / 童装）依赖服饰图信号（版型、童趣元素等）；信号弱或中性款时可能误判年龄段，建议用 `imageDesc` 显式锁定。
- 多张参考图时由模型自行判断服饰图 / 模特参考图 / 场景参考图的角色，复杂组合下可能误判主图。
- 本类型只做从服饰图"生成"模特上身图，不做对已有模特图的换姿势 / 换动作裂变（模特裂变为独立能力，提示词由业务方上游构造，不读字典）。
