# S1 内容推理

- **输入**：图片 + point/pointHint/imageDesc（agent 层传入）+ 上下文历史卖点
- **依赖**：S0（场景路由决定是否执行本步）
- **操作**：识别品类/材质/形状，推理卖点与画面内容
- **输出**：point[] / image_desc[] / desc[] / layout[]
- **用途**：作为 S2 规划方案（imagePlanList）的内容来源

---

## 并行执行：S1 内容推理 + 品牌基因提取

S0 场景路由完成后，**在同一轮响应中同时触发**以下两个任务，两者都完成后才进入 S2：

| 并行任务 | 调用方式 | 条件 |
|----------|----------|------|
| S1 内容推理 | 当前步骤（见下方规则） | 场景路由要求执行步骤一时 |
| 品牌基因提取 | 调用 skill `linkfox-aigc-imagegen-brand-gene-extract` | 仅当上下文中**没有**已提取的品牌基因时（即场景D首次做套图） |

**品牌基因提取参数组装（仅在需要提取时）：**
- `images`：用户上传的商品图片
- `brandKey`：组装方式——
  - 如果用户输入中包含 `品牌基因` 字段，直接使用其中的 language、platform、salesRegion；若包含 brandColor、fontStyle 一并传入
  - 如果用户输入中不包含 `品牌基因` 字段，则从用户意图或上下文中提取 language、platform、salesRegion
  - 若从意图/上下文中均无法提取，默认填充：salesRegion=美国、language=英文、platform=亚马逊
- skill 返回的 JSON 即为 `brandGeneJson`；**必须**经 `linkfox-aigc-imagegen-brand-gene-extract` 的 `save_brand_gene.py` 落盘到会话 `data/`，将 stdout 的 `Saved full response: <绝对路径>` 记入上下文，供后续 plan 阶段写入 `collection-state.json` 的 `brand_gene_file`、run_one_task 据此传**文件路径**复用（**禁止**内联进 state、**禁止**仅存于对话上下文）

**不触发品牌基因提取的场景（B/C/E/F）**：上下文中已有品牌基因结果，直接复用，仅执行 S1 内容推理。

## 输出体积控制

S1 的推理结果（point / image_desc / desc / layout）是 S2 imagePlanList 的**内部素材**，不是面向用户的展示内容。执行约束：

1. **禁止逐条向用户展示** S1 推理的 point、image_desc、desc 原文——它们只需内部持有，直接压缩进 S2 的 imagePlanList 规划表中呈现
2. S1 完成后**立即进入 S2 组装 imagePlanList**，不做中间汇报，以控制单轮响应体积、避免因响应过长被截断
3. S2 用户确认后须按 `S2-plan-confirm.md`「用户确认后立即落盘」经 `save_image_plan.py` 逐条写入 `$DATADIR/image-plan.json`，**禁止**把完整 plan 只留在上下文供 S3 重建

## 输入完整性判断

在进入步骤一之前，根据 agent 层传入的结构化字段判断推理深度：

**卖点判断：**
- `point` 非空 → 直接使用该值作为对应图片的卖点，跳过该张图的卖点推理
- `point` 空 + `pointHint` 非空 → 以 hint 为方向约束进行卖点推理（推理结果须围绕 hint 方向）
- `point` 空 + `pointHint` 空 → 全自动推理卖点

**画面内容判断：**
- `imageDesc` 非空 → 直接使用该值作为对应图片的 image_desc，跳过画面内容推理
- `imageDesc` 空 → 自动推理画面内容

**多张图场景：**
- 传入 N 个 point，需要 M 张卖点相关图（M > N）→ 前 N 张用传入值，剩余自动推理（遵守去重规则）
- 传入 1 个 point，需要多张图 → 第 1 张用传入值，其余自动推理

---

## 卖点不重复规则

- 本次推理的卖点必须与**上下文中所有历史轮次**已使用过的卖点都不重复
- 同一批次内，同一类型的图片之间卖点也不重复

---

## 卖点(point)推理

point 是传递给共享流水线的核心视觉描述，必须是一段**详细的卖点描述词**，绝不能只是简短的关键词标签。

**point 推理要求**

1. **必须使用中文**，无论面向何种市场，point 描述本身始终输出中文
2. 根据「输入完整性判断」的结果决定推理策略：`point` 非空时直接使用、`pointHint` 非空时围绕 hint 方向推理、都为空时全自动推理。如果与卖点相关的作图数比已有 point 多，就根据已有 point 做相似的卖点裂变，改变描述，但不改变意思。
3. 必须与上下文中历史所有轮次已使用过的卖点不重复，同一批次内同类型图片之间卖点也不重复（允许相同卖点的不同描述）
4. 每个 point 严格控制在 **2~3 句话**，不能只写简短的关键词
5. 推理的卖点个数与确定后的套图中卖点有关的图片数相同
6. 商品有多个的时候，卖点可以按单件或套装来分别推理
7. 卖点提取维度：功能、性能、材质/成分、工艺/技术、外观/颜值

**point 示例**

**卖点图示例**：
> 双层罗纹收口搭配高弹松紧腰头，久坐不紧勒，运动不下滑，日常通勤到健身房无缝切换。

**种草图示例**：
> 轻薄透气的冰丝面料，自带UPF50+防晒指数，夏日户外穿着凉爽不闷汗，一件搞定防晒与时尚。

## 画面内容(image_desc)

image_desc 是根据商品和对应的卖点，设计和推理出面向用户展示的预期的图片内容，画面内容需与卖点和作图类型严格对应，且符合商品和品牌调性。

- 必须使用中文，限制在**100个中文字符**以内
- 不允许直接描述原商品图中的场景内容
- 画面内容为2~4 句话的详细视觉描述，包含画面主体、镜头角度、光线氛围、背景环境等具体信息，不能只写简短的关键词
- 当作图类型为种草图（SCENE）时，自动分析该商品的目标受众和商品的风格，推理出用于激发用户购买欲望的多种Ins风格场景，姿势与穿搭方案（必须有真人穿着该商品或多件商品）。
- 人物说明：
1. 画面中包含人物时，默认是欧美白人，除非用户特殊说明，否则不允许出现黑人。
2. 人物覆盖原则： 用户提供的输入信息具有最高优先级。若明确为"儿童"，则 image_desc 中严禁出现"成年"、"男性/女性模特"等词汇，画面内容必须严格围绕儿童来描述。
- 当作图类型为模特图（MODEL_IMAGE）时，需要根据上传的图片内容和用户输入指令，自动判断，是否需要参考某张图中的模特或场景。
  a. 无参考模特或场景：需要根据商品的信息（设计元素，版型比例），自动分析年龄与性别，推导潜在买家的年龄、地域肤色偏好、消费场景（是为了"去海边度假"、"办公室通勤职业/商务"、"健身房打卡"、"户外机能风"、"居家High Tea"、"日常休闲"、"内衣睡衣私密卧室场景"还是"婚纱、万圣节等特殊场合等），场景遵循高端商业场景，专业相机拍摄，既有新意又符合Amazon/Shein调性。除非用户特殊说明极简或纯色背景，否则，场景禁止为纯色或极简。模特必须为单人,服装为正面时，禁止描述背面朝向。
  b. 有参考模特或场景：生成的画面内容中的人物和场景内容必须按照参考图的模特或场景来描述。例如，参考图是一个女性，则描述为："一个欧美金色短发年轻女模特..."。再例如参考图是一张草地的图，则描述为："在大片绿色的草地上..."，禁止出现"参考图"等字眼。
  注意：
1. 姿势设计：正面站立为主，双手自然下垂，保证商品（如上衣或下衣）完全展示，必须不遮挡核心细节，眼神与镜头交流，确保头身比自然。
2. 模特图的画面内容（image_desc）描述格式示例为："一个xxx模特，穿着xxx，xx姿势，xx景别，站在xxxx场景，氛围"
- 当作图类型是尺码图（SIZE）时，画面内容固定为："这是一张商品尺码示意图，图中详细标注了产品的各项关键参数"
- 白底图不需要画面内容

**image_desc 示例**

**卖点图示例**：
> 服装平铺在浅色木桌上，镜头聚焦腰部松紧带细节和罗纹收口工艺，自然光侧打，突出面料的弹性质感和精细车缝线。

**种草图示例**：
> 一位欧美年轻女性穿着该连衣裙，漫步在阳光斑驳的南法小镇石板路上，手提草编包，裙摆随微风自然飘动，营造浪漫度假氛围。

## 简单描述(desc)

desc 是面向用户展示的简短中文主题标签，是对画面内容（image_desc）的简述，用于在规划方案中快速标识每张图的主题。

- 必须使用中文
- 一般类型（卖点图、种草图、尺码图等）限制在 **10个中文字符**以内
- 模特图需要简述画面+姿势+景别，限制在 **15个中文字符**以内
- 白底图不需要简单描述

**desc 示例**

**卖点图示例**：
> 弹力腰头，久坐不勒

**种草图示例**：
> 南法度假，裙摆飘逸

**模特图示例**：
> 街头漫步侧面全身自然

## 版式信息(layout)

layout 是卖点图、A+图、尺码图的画面排版（版式）。不同图片类型有各自独立的版式规则：

**卖点图 / A+ 图版式**：版式由外部传入，按各自适合的场景风格打上不同标签：概览、功能、品质、场景。
- 假如有多张卖点图/A+图，则每张图对应的版式需按照: **概览 → 功能 → 品质 → 场景** 的优先级**轮流**选择。例如：卖点图1 选择 概览，卖点图2 选择 功能...
- **卖点图/A+图的 layout 只能从 概览、功能、品质、场景 四个标签中选择，严禁使用其他自定义值**

**尺码图（SIZE）版式**：固定为 **普通** 和 **三版** 两种，按 **普通 → 三版** 的顺序**轮流**选择。
- 例如：尺码图1 选择 普通，尺码图2 选择 三版，尺码图3 选择 普通...
- **尺码图的 layout 只能从 普通、三版 中选择，严禁使用其他值**

除卖点图、A+图、尺码图之外，其他图片类型 layout 传空字符串。
